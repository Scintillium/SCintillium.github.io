    二叉樹的非遞歸遍歷

    算法 leetcode 二叉樹
    Created At : 2022-05-18 10:32
    Count:2k Views 👀 :
    Comment:
    1. 問題描述
    2. 遞歸解法
    3. 非遞歸解法

    二叉樹的遍歷算是非常常見，非常基礎的算法題，也有很多題是根據各種樹的遍歷而改編過來的。二叉樹的遍歷，可以通過簡單的遞歸方法來實現。然而有些面試官會別出心裁，會來問一下用非遞歸的方式，是如何來實現。說實話第一次遇到這中問題確實把我難住了，明明是常見的問題，卻突然變的十分陌生，不知從何下手。

    問題描述

    鏈接：

    • leetcode 94. 二叉树的中序遍历
    • leetcode 144. 二叉树的前序遍历
    • leetcode 145. 二叉树的后序遍历

    給定一個二叉數，其結點的結構爲

    struct TreeNode {
    int val;
    TreeNode *left;
    TreeNode *right;
    TreeNode() : val(0), left(nullptr), right(nullptr) {}
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
    TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
};

    然後再給你二叉樹的根結點root，請返回它結點值的前序遍歷/中序遍歷/後序遍歷。

    遞歸解法

    以中序遍歷爲例，在中序遍歷函數內調用遞歸函數traversal()

    vector<int> inorderTraversal(TreeNode* root) {
    vector<int> res;  //保存遍歷序列
    traversal(root, res); 
    return res;
}

    其中中序遍歷的traversal()的寫法爲：

    void traversal(TreeNode* cur, vector<int>& res) {
    if (cur == nullptr) return; // 終止
    traversal(cur->left, res);  // 左子樹
    res.push_back(cur->val);    // 中間結點
    traversal(cur->right, res); // 右子樹
}

    舉一反三，可以得到前序遍歷的traversal()的寫法爲：

    void traversal(TreeNode* cur, vector<int>& res) {
    if (cur == nullptr) return; // 終止
    res.push_back(cur->val);    // 中間結點
    traversal(cur->left, res);  // 左子樹
    traversal(cur->right, res); // 右子樹
}

    後序遍歷爲：

    void traversal(TreeNode* cur, vector<int>& res) {
    if (cur == nullptr) return; // 終止
    traversal(cur->left, res);  // 左子樹
    traversal(cur->right, res); // 右子樹
    res.push_back(cur->val);    // 中間結點
}

    非遞歸解法

    在遞歸的情況下，程序需要借助棧來實現函數調用。每次調用遞歸函數都會將參數值和返回地址壓入棧中。因爲棧遵循First in Last out， 故而遞歸函數執行完之後可以返回原來的位置。
    顯然用非遞歸的方式實現二叉樹的遍歷需要顯示地借助於棧這個數據結構來實現函數調用棧的過程。

    這樣一想這個問題其實並不難，前序遍歷可以用如下的方式實現：

    vector<int> preorderTraversal(TreeNode* root) {
        stack<TreeNode*> stk;       //借助棧來實現遍歷
        vector<int> res;
        if (root == bullptr) return res;
        st.push(root);
        TreeNode * node;
        while (!st.empty()) {
            node = st.top(); st.pop();
            res.push_back(node->val);           //記錄根結點
            if (node->right) st.push(node->right);  //將左子結點壓入棧
            if (node->left)  st.push(node->left);   //將右子結點壓入棧
        }
        return res;

    然後，我們突然發現這種寫法似乎只適用於前序遍歷了，如果按照遞歸的慣性思維, 將上述代碼的循環部分改爲

                if (node->right) st.push(node->right);  
            res.push_back(node->val);  
            if (node->left)  st.push(node->left);

    來作爲中序遍歷的實現，又或者是

                if (node->right) st.push(node->right);  
            if (node->left)  st.push(node->left);
            res.push_back(node->val);

    作爲後序遍歷的實現。然而運行程序會發現返回的結果都是前序遍歷的結果。爲什麼會這樣呢？

    我們再來重新回顧一下中序/前序/後序遍歷的遞歸實現，看看問題出在哪裏。首先在遞歸思想當中，將整個問題歸納爲幾個重複的子過程，將這個過程封裝爲函數重複調用。以中序中序遍歷爲例，它可以看做 L（中序遍歷的左半部分）+ M（根結點）+ R（中序遍歷的右半部分）。一般解釋爲先遍歷左半部分，然後再遍歷根結點，再遍歷右半部分。這種解釋雖然雖然正確，但是含混不清。必須指出的是，這種先後順序是指時間上的順序。我們來看中序遍歷的遞歸實現

     void traversal(TreeNode* cur, vector<int>& res) {
    if (cur == nullptr) return; // 終止
    traversal(cur->left, res);  // 左子樹
    res.push_back(cur->val);    // 中間結點
    traversal(cur->right, res); // 右子樹
}

    在一次調用traversal當中，如果沒有遇到終止條件，traversal(cur->left, res);執行完成時，L就已經完成了，然後再添加M，再執行traversal(cur->left, res);時R也全部完成了，最終結果就是L+M+R。然後我們再來看一開始前序遍歷的非遞歸實現的循環部分：

    while (!st.empty()) {
    node = st.top(); st.pop();
    res.push_back(node->val);           //1
    if (node->right) st.push(node->right);  //2
    if (node->left)  st.push(node->left);   //3
}

    這個循環記錄了每次循環中從棧中彈出的元素，也意味着，無論如何調換語句1，2，3的順序，這次循環只會記錄此次的根結點，而左右子結點只會在後面的循環當中被記錄，根結點訪問時間先於左右子結點，故而無論如何調換順序都只會得到前序遍歷。所以試圖實現中序遍歷的話，要確保訪問記錄根結點M的時候，L已經完成遍歷，訪問M之後當然就是只剩下R的遍歷。

    無論是遞歸實現還是非遞歸實現，樹的遍歷過程都可以視爲一個深度優先搜索（DFS）的過程。爲了實現中序遍歷，我們必須顯示地執行完其左邊的遍歷過程，而非將所有結點都不分左右地放入棧中。例如上述的中序遍歷的非遞歸實現，每次從棧中彈出的結點，處理上都是相同的，我們也無從判斷其在遍歷過程當中所處的位置，不能區分左子樹和右子樹，所以就不能在合適的時機記錄中序遍歷的結點。中序遍歷的非遞歸實現如下：

    vector<int> inorderTraversal(TreeNode* root) {
    stack<TreeNode *> stk;
    vector<int> res;
    TreeNode * cur = root;
    while(!stk.empty() || cur){
        while(cur){      //最左深度優先，全部壓入棧中
            stk.push(cur);
            cur = cur->left;
        }
        if(!stk.empty()){ 
            cur = stk.top(); stk.pop(); //此時出棧的元素
            res.push_back(cur->val);    //此時其左子樹已經全部遍歷完成
            cur = cur->right;           //壓入右子樹
        }
    }
    return res;
}

    上面我們說到，其實三種遍歷都是DFS，所以前序遍歷也能應用上面的代碼，只不過需要改變記錄結果的位置。中序遍歷記錄的時機是其左邊所有結點都已經出棧的時候，而前序遍歷則是記錄在每個結點入棧的時候，所以代碼實現如下：

    vector<int> inorderTraversal(TreeNode* root) {
    stack<TreeNode *> stk;
    vector<int> res;
    TreeNode * cur = root;
    while(!stk.empty() || cur){
        while(cur){     
            stk.push(cur);
            res.push_back(cur->val);   //此時記錄可以保證其子孫結點都出現在cur的後面
            cur = cur->left;
        }
        if(!stk.empty()){ 
            cur = stk.top(); stk.pop();  
            cur = cur->right;           
        }
    }
    return res;
}

    那麼後序遍歷是如何實現呢？我們知道前序遍歷是M+L+R，其實我們將前序遍歷的最左深度優先改爲最右深度優先，得到的序列將是M+R+L。同時我們也知道後序遍歷爲L+R+M，正好和M+R+L逆序，所以後序遍歷可以這樣實現：

    vector<int> inorderTraversal(TreeNode* root) {
    stack<TreeNode *> stk;
    vector<int> res;
    TreeNode * cur = root;
    while(!stk.empty() || cur){
        while(cur){     
            stk.push(cur);
            cur = cur->right;  //最左改成最右
            res.push_back(cur->val); 
        }
        if(!stk.empty()){ 
            cur = stk.top(); stk.pop();   
            cur = cur->left;           
        }
    }
    reverse(res.begin(), res.end()); //翻轉
    return res;
}

    至此三種遍歷的非遞歸實現得到了一種統一的寫法。
